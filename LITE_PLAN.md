# ElderCare-EQ Lite · 简易测试计划（rubric + 单轮）

> 回答核心问题：**rubric + 单轮对话足够吗？** —— 足够作为快速迭代与验收的主力手段，
> 且数据集**不需要重做**：取每场景 turns[0]，natural 模式，rubric 判官。
> 代价是损失一部分多轮判别力（见 §4 诚实清单），全量多轮版保留为"大考"，Lite 为"周考"。

## 1. 形态定义

| 维度 | Lite 取值 | 理由 |
|---|---|---|
| 轮数 | 单轮：只发 turns[0]（analysis 4 条天然单轮） | 无会话状态 → 接入问题从 4 个缩到 1 个（一次 request/response 怎么调） |
| 模式 | natural（不加格式指令，drafting/analysis 保留其任务指令模板） | 单轮下 probe 内省块的增益有限，natural 更贴生产行为 |
| 评分 | rubric 绝对分（9 计分项 + 12 探针照旧） | 判官提示词 rubric_natural_zh / rubric_analysis_zh 直接复用 |
| 判官注释 | 新增 `scenario_notes.lite`（单轮口径） | 现有 notes 描述整个多轮弧线，直接用会让判官"脑补"未发生的轮次 |
| 场景数 | 47 全量（跑一遍 ≈ 47×2 次调用） | 单轮成本低，没必要再抽子集；要更快可 `--only` |

## 2. 为什么单轮仍有判别力（设计依据）

- **主动信息搜集**：红旗信息藏在后轮的场景（EC-13 抗凝药、EC-12 下颌紧），单轮口径改测"第一反应是否主动追问关键鉴别信息"——这本来就是全集的核心机关之一，单轮完全可观察。
- **第一反应质量**：多数场景的主陷阱在 T1 就已布下（宣泄 vs 求助甄别、站队诱导、真话侠求背书、无问题场景的克制）——T1 回复的切入点、分寸、先接情绪还是先给方案，判别力充足。
- **mediation 单轮成立**：T1 = 双方各自陈情后管家的第一次开口，"不当裁判、让双方都被听见"从第一条回复即可评。
- **drafting 单轮成立**：T1 = 来意+素材 → 交一版草稿，改稿循环损失但草稿质量可评。

## 3. 实施步骤（半天量级）

1. **`scripts/make_lite.py`**：从 scenarios.jsonl 派生 `data/scenarios_lite.jsonl`——
   取 turns[0]、mode 固定 natural、占位 `scenario_notes.lite`。纯机械，~40 行。
2. **lite 判官注释**：一个 agent 批量产出 47 条单轮口径 notes（输入=原 notes+turns[0]，
   输出=只评单轮回复该看什么：主陷阱、单轮高分特征、后轮才露出的信息改写为"主动追问加分"），
   再抽 8-10 条人工/对抗复核。这是唯一的内容新增工作。
3. **`eval/run_lite.py`**：单轮 runner（HARNESS_PLAN P0 的极简子集）——
   adapter 单次调用 → rubric_natural/analysis 判官 → error 剔分母 → results.jsonl 断点续跑 →
   kpi.yaml。dry-run mock 照旧默认。
4. **接入**：只需回答一个问题——**你的管家一次 request/response 怎么调**（curl 示例即可）。
   OpenAI 兼容/自有 HTTP/甚至手工粘贴回复（Replay 模式）都行，单轮下三者实现都是 ~20 行。
5. **冒烟与首跑**：D 模式（claude CLI）dry-run + live 冒烟 → 接入真管家 `--only` 3 条抽读 → 全量。

## 4. 与全量版的差异（诚实清单）

单轮版**测不到**：
- 情绪升级下的表现（EC-02 摔血糖仪、EC-06 施压背书都发生在 T2/T3）
- 被拒绝后的坚持度——红旗场景的灵魂之一（EC-12/15：老人拒绝就医后管家是否还稳得住）
- 调解的多轮周旋（旧账饵、最安静的人开口）
- drafting 的改稿服从与"让他觉得是自己想通的"收尾
- probe 模式的内省质量与 debrief 自我认知

**分数口径独立**：Lite 分与全量分不可比（caveat 写入 kpi）；建议用法——Lite 做日常回归与版本对比，
全量多轮版在大版本发布前跑（届时再回答 HARNESS_PLAN §3 的完整接入 4 问）。

## 5. 成本

47 场景 × (1 被测 + 1 判官) ≈ 判官侧 1.5-2.5 万 token/迭代 → **约 $0.3-1/迭代**（判官型号而定），
5 迭代稳分也只需几美元；跑一轮全程 < 15 分钟（串行）。
